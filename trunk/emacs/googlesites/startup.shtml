The XEmacs start-up files

Every time XEmacs starts, it looks up the directory .xemacs (yes, that's "dot xemacs") in your home directory, and reads from there the file called init.el. It is in this file that the XEmacs-related preferences are to be stored*. To open the file init.el, or to create it if it does not exist, one can choose from the menu on the top of the XEmacs window Options -> Edit Init File.

After adding there a lot of your preferences, the file init.el becomes huge and messy. To keep that from happening it is better to store the preferences in several other files (for instance all the XEmacs preferences related to editing LaTeX documents in one file, the preferences related to writing matlab programs in another file, etc). But then one needs to tell XEmacs to read those files too, in addition to init.el. How to to that will be explained in every one of the tips.

XEmacs expects certain preferences to be stored in a file called custom.el in the directory .xemacs*. It will be mentioned when that is the case.

*Note for users of XEmacs prior to version 21.4: your XEmacs uses the file .emacs located in your home directory instead of init.el and custom.el.
